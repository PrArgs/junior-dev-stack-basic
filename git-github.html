Git & GitHub

Version control and collaboration

What are Git and GitHub?

Git is a distributed version‑control system that records snapshots of your code over time. You work in a local copy of your repository, stage changes with git add and record a snapshot (a commit) with git commit. Git also lets you branch and merge your project history.

GitHub is a cloud platform that hosts Git repositories and adds collaboration features. GitHub’s pull‑request workflow helps teams propose, review and merge changes. GitHub’s pull‑request workflow

GitHub’s quick‑start guide outlines these steps:

  1. Create a repository – A repository is like a project folder where you store files. When creating a new repository you can include a README file and choose whether it is public or private【778979557977785†L185-L199】.
  2. Create a branch – Branching lets you work on new features independently of the main code. You can create additional branches off of main and later merge them back【778979557977785†L224-L233】.
  3. Make and commit changes – Edit files and then save your changes as a commit. GitHub displays your changes and asks you to write a commit message explaining what you did【778979557977785†L274-L289】.
  4. Open a pull request (PR) – A PR proposes your changes to others, shows the differences between branches and allows for discussion and reviews【778979557977785†L291-L306】. When the PR is ready, you can merge it into main【778979557977785†L309-L325】.

Git basics: staging, committing and remotes

Git stores changes in a three‑stage process: the working directory, the staging area and the commit history. You stage changes with git add and commit them with git commit. The commit message should explain the changes【67246901121458†L626-L689】. To collaborate, you define remote repositories; when you clone from GitHub, the remote is called origin. You can list remotes with git remote -v and add new ones with git remote add <name> <url>【906919906535863†L205-L224】.

Hands‑on example

Below is a simple sequence of commands you can run in a terminal to create a new Git repository, add a file and commit it. Try these steps in a practice directory on your own machine. 

# Create a new folder and initialize a Git repository
mkdir demo_repo
cd demo_repo
git init

# Create a README file
echo "Hello Git!" > README.md

# Stage and commit the file
git add README.md
git commit -m "Initial commit"

# View the commit history
git log --oneline
